AIエージェントプロトコルの時代

2024年から2026年にかけて、AIエージェントの相互運用性を実現するための様々なオープンプロトコルが提唱されてきました。これらのプロトコルは、それぞれ異なる目的と設計思想を持ち、AIエージェント・エコシステムの発展に貢献しています。

MCP (Model Context Protocol)

概要

Model Context Protocol (MCP)は、LLMアプリケーションと外部データソース・ツールをシームレスに統合するためのオープンプロトコルです。Anthropicが2024年11月に発表し、現在はAAIF（Agentic AI Foundation）の傘下で管理されています。

MCPは、AIエージェントに「手」を提供するプロトコルとして位置づけられており、エージェントが外部のツール、データベース、APIを標準化された方法で利用できるようにします。

主な機能

Resources (リソース)

ファイル、データベースレコード、APIレスポンスなど、エージェントが読み取り可能なデータソースを提供します。

Prompts (プロンプト)

再利用可能なプロンプトテンプレートを定義し、一貫性のあるユーザー体験を提供します。

Tools (ツール)

エージェントが実行できる関数やAPIを公開し、外部システムとのインタラクションを可能にします。

Sampling (サンプリング)

サーバー側でLLMを呼び出し、エージェントフローやチェーン処理を実装します。

アーキテクチャ

設計思想: クライアント・サーバー型アーキテクチャ

通信プロトコル: JSON-RPC 2.0

トランスポート: stdio（標準入出力）、SSE（Server-Sent Events）、HTTP

設定例 (mcp.json)

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "@modelcontextprotocol/server-filesystem",
        "/Users/yourname/Documents"
      ],
      "env": {
        "API_KEY": "${YOUR_API_KEY}"
      },
      "disabled": false,
      "autoApprove": ["read"]
    },
    "postgres": {
      "command": "npx",
      "args": [
        "@modelcontextprotocol/server-postgres",
        "postgresql://localhost/mydb"
      ]
    }
  }
}

A2A (Agent-to-Agent Protocol)

概要

Agent-to-Agent Protocol (A2A)は、AIエージェント間のコミュニケーションと相互運用性を実現するオープンプロトコルです。Googleが2025年4月に発表し、Linux Foundation傘下で管理されています。

A2Aは、AIエージェントに「声」を提供するプロトコルとして位置づけられており、異なるベンダーや組織が開発したエージェント同士が標準化された方法で対話・協働できるようにします。

主な特徴

  • ピアツーピアアーキテクチャ: エージェント同士が対等な立場で直接通信
  • マルチプロトコル対応: HTTP、SSE（Server-Sent Events）、JSON-RPCをサポート
  • Agent Card: エージェントの能力・仕様を記述した標準化された自己紹介カード
  • 柔軟な通信: 同期・非同期、リクエスト・レスポンス、ストリーミングに対応
  • セキュリティ: 認証・認可メカニズムを標準実装

Agent Card例

{
  "name": "seller-agent",
  "description": "遠隔販売エージェント",
  "version": "0.3",
  "service_endpoint": "https://example.com/agent",
  "supported_modalities": ["text", "json"],
  "capabilities": [
    {
      "name": "product_search",
      "description": "商品検索機能",
      "input_schema": {
        "type": "object",
        "properties": {
          "query": { "type": "string" },
          "category": { "type": "string" }
        }
      }
    }
  ],
  "authentication": {
    "type": "bearer_token",
    "token_endpoint": "https://example.com/oauth/token"
  },
  "rate_limits": {
    "requests_per_minute": 60
  }
}

ACPとの統合

統合時期: 2025年9月より統合開始

背景: IBM BeeAIが開発したAgent Communication Protocol (ACP)は、類似の目的を持つプロトコルでした。両チームは協議の結果、エージェント通信プロトコルの分断を避けるため、ACPの技術的貢献をA2Aに統合する方針を決定しました。

現状: ACPは独立したプロトコルとしての開発を終了し、A2Aへの移行が進められています。ACPの優れた設計要素はA2A v0.3以降に取り込まれています。

UCP (Universal Commerce Protocol)

概要

Universal Commerce Protocol (UCP)は、エージェント型コマースのための次世代オープンソース標準です。Googleが2026年1月11日にNRF Retail's Big Showで発表しました。

UCPは、小売・EC事業者が既存のインフラを活用しながらAIエージェントに対応できるよう設計されており、REST API、MCP、A2Aの3つのプロトコルを柔軟に統合できる点が特徴です。

主な特徴

統一言語

商品検索、カート操作、決済、配送など、EC機能を標準化されたボキャブラリーで定義します。

柔軟な統合

REST API、MCP、A2Aのいずれかまたは複数を選択して実装できます。

既存インフラ互換

既存のECプラットフォームやシステムをそのまま活用できる設計です。

      }

      "agent_card": "https://example.com/.well-known/a2a"
    }
  },
  "capabilities": [
    {
      "name": "product_search",

    },

アーキテクチャの柔軟性

UCPは単一のプロトコルではなく、3つの実装方式を提供しています。

  • REST API実装: 従来型のHTTP APIとして実装。既存システムとの統合が容易。
  • MCP実装: MCPサーバーとして実装。LLMアプリケーションとの統合に最適。
  • A2A実装: Agent Cardとして実装。エージェント間連携に最適。

事業者は自社のシステム構成とニーズに応じて、1つまたは複数の方式を選択できます。

Agent Skills

概要

Agent Skillsは、AIエージェントに新しい能力と専門知識を与えるオープンフォーマットです。Anthropicが2025年10月に発表し、現在はAAIF（Agentic AI Foundation）の傘下で管理されています。

スキルは、エージェントが学習・実行できるタスクや操作を標準化されたフォーマットで定義します。エージェントは必要に応じてスキルをロード・実行することで、能力を拡張できます。

SKILL.md構造例

---
name: pdf-skill
description: PDFドキュメントの読み取り、フォーム入力、抽出を行うスキル
version: 1.0.0
author: Example Corp
tags: [pdf, document, extraction]
dependencies: []
---

# PDF Skill

このスキルはPDFファイルの操作に特化しています。

## 機能

- PDFファイルの読み取り
- テキスト抽出
- フォーム入力
- メタデータ取得

## 使用例

PDFファイルからテキストを抽出する場合:
```
extract_text_from_pdf("path/to/file.pdf")
```

## 前提条件

- PyPDF2ライブラリがインストールされていること
- Python 3.8以上

ディレクトリ構造

my-skill/
├── SKILL.md          # メインのスキル定義ファイル（必須）
├── reference.md      # API仕様・リファレンス
├── examples.md       # サンプルコード・ユースケース
├── scripts/          # 実行可能なスクリプト
│   ├── setup.sh
│   └── main.py
└── assets/           # 画像・設定ファイル等
    └── config.json

スキルの特徴

再利用可能

一度定義すれば、複数のエージェント・プロジェクトで再利用できます。

組み合わせ可能

複数のスキルを組み合わせて、より複雑なタスクを実行できます。

バージョン管理

スキルはバージョン管理され、依存関係も明示的に定義できます。

コミュニティ駆動

オープンソースとして公開され、コミュニティが継続的に改善します。

SaaS・プラットフォーム: Canva、Stripe、Notion、Zapier、Microsoft、OpenAI

開発ツール: Atlassian、Figma、Cursor、GitHub

備考: 多くの企業が自社サービス向けのスキルを公開し、エージェント・エコシステムの拡大に貢献しています。

AGENTS.md

概要

AGENTS.mdは、コーディングエージェント向けのREADMEです。OpenAIが2025年8月に提唱し、現在はAAIF（Agentic AI Foundation）の傘下で管理されています。

AGENTS.mdは、AIコーディングエージェントがプロジェクトで作業するためのコンテキストと指示を提供する標準化された場所として機能します。人間の開発者にとってのREADME.mdと同様の役割を果たします。

AGENTS.md例

エコシステムと将来展望

Agentic AI Foundation (AAIF)

組織概要

設立: 2025年12月

母体: Linux Foundation

使命: エージェントAI技術のオープンスタンダード化とエコシステム構築

主要プロジェクト:

  • MCP (Model Context Protocol) - Anthropic寄贈
  • AGENTS.md - OpenAI寄贈
  • goose - Block寄贈
  • Agent Skills - Anthropic寄贈

サポーター企業: Microsoft、AWS、Cloudflare、Bloomberg、その他多数

プロトコル間の関係性

MCP + A2A = 完全なエージェントシステム

MCPとA2Aは相補的な関係にあります。

  • MCP (手): エージェントがツール、データ、APIを使えるようにする
  • A2A (声): エージェント同士がコミュニケーションできるようにする

両者を組み合わせることで、ツールを使いこなし、他のエージェントと協働できる完全なエージェントシステムを構築できます。

ACP → A2A統合

IBM BeeAIが開発したACPは、2025年9月にA2Aチームとの統合を発表しました。エージェント通信プロトコルの分断を避け、業界全体の相互運用性を高めるための決定です。ACPの技術的貢献はA2Aに統合され、独立したプロトコルとしての役割は終了しています。

Agent Skills + MCP

Agent SkillsとMCPは異なる目的を持ちながら、互いに補完し合います。

  • Agent Skills: エージェントに「知識」を提供（タスクの実行方法を定義）
  • MCP: エージェントに「手」を提供（実際のツール実行を可能にする）

スキルがタスクの手順を定義し、MCPが実際のツール実行を担当する形で連携します。

UCP: 領域特化型統合

UCPはコマース領域に特化しながら、REST API、MCP、A2Aのいずれとも統合できる柔軟な設計です。既存のECインフラを活用しつつ、エージェント対応を実現できます。

AGENTS.md: 独立した規約

AGENTS.mdは他のプロトコルとは独立した規約です。プロジェクトのコンテキストをエージェントに提供する標準的な方法として機能します。

2026年の展望

1. 標準化の加速

AAIFの設立により、オープンスタンダードの策定と普及が加速します。業界横断的な合意形成が進み、相互運用性が向上します。

2. プロトコル統合

ACP→A2A統合のように、類似プロトコルの統合が進みます。分断を避け、開発者の負担を軽減する動きが続きます。

3. 役割分担の明確化

MCP（ツール統合）、A2A（エージェント通信）、Agent Skills（能力拡張）、AGENTS.md（コンテキスト提供）という役割分担が明確になり、各プロトコルが適切な領域で発展します。

4. エンタープライズ採用の拡大

大手コンサルティング企業（Accenture、BCG、Deloitte、McKinsey、PwC）の参画により、エンタープライズ領域での採用が加速します。

5. 開発者エコシステムの成長

MCPの97M月間DL、AGENTS.mdの60K+プロジェクト採用など、開発者コミュニティが急速に拡大しています。ツール、ライブラリ、スキルの共有が活発化します。

6. 領域特化型プロトコルの登場

UCPに続き、金融、医療、製造など他の業界でも領域特化型プロトコルが登場する可能性があります。
